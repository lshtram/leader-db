# Documentation Index

This folder is organized by documentation type. Keep only a small number of navigational/status documents at the top level; put substantive project documents in the category folders below.

## Start here

- [`workplan.md`](workplan.md) — current status, active phase, blockers, and done history.
- [`viz-workplan.md`](viz-workplan.md) — visualization-layer plan and Superset/semantic-query decisions.
- [`requirements/top-level-requirements.md`](requirements/top-level-requirements.md) — authoritative product brief.
- [`architecture/overview.md`](architecture/overview.md) — system design and component boundaries.

## Folders

- [`architecture/`](architecture/) — architecture overview, source-system architecture, database schema, local data-store rules, and source-specific implementation designs.
- [`requirements/`](requirements/) — top-level, core, and source-system requirements.
- [`sources/`](sources/) — source registry, attribution registry, historical ingestion plan, source-system guide, and vetting materials.
- [`process/`](process/) — coding guidelines and operational hygiene rules.
- [`chronicle/`](chronicle/) — Country-Year Chronicle plans and increment notes.
- [`methodology/`](methodology/) — ranking and evaluation methodology.
- [`reviews/`](reviews/) — reviewer gate outputs.

When moving or adding documents, update this index plus any links in `AGENTS.md`, `README.md`, and `docs/workplan.md` that point to the affected files.
