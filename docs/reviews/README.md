# Reviews

This folder holds reviewer outputs.

## How to use

When the user asks for **TDD** or a formal review gate, write the reviewer decision here as `docs/reviews/<date>-<module-or-slug>.md` with the following sections:

- **Summary** — one paragraph
- **Decision** — `approved_for_acceptance` / `requires_iteration` / `impossible_to_improve`
- **Blockers** — list of issues that must be fixed before the next phase
- **Done when** — concrete acceptance criteria
- **Non-blockers** — improvements that can be deferred
